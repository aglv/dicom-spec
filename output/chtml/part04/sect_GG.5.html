GG.5 Conformance Statement Requirements

An implementation may conform to any of the Non-Patient Object Storage SOP Classes as an SCU, SCP or both. The Conformance Statement shall be in the format defined in PS3.2.

GG.5.1 SCU Conformance Requirements

An implementation that conforms to a SOP Class of the Non-Patient Object Storage Service as an SCU shall state in its Conformance Statement:

  • Whether the implementation is a SOP Instance creator for the SOP Class.

    Note

    There may be SOP Class specific Conformance Statement requirements for creators of SOP Instances. See Section GG.6.

  • The behavior of the SCU in the case of a success C-STORE response status.

  • The behavior of the SCU in each case of a failure C-STORE response status.
